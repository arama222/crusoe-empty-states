Crusoe Cloud offers three types of disks: Ephemeral Disks, Persistent Disks, and Shared Disks. Ephemeral disks are local to the VM for highest performance, persistent disks provide high availability from 1 GiB to 10 TiB, and shared disks support concurrent multi-VM access via NFS.
Documentation
Remote block storage volumes with high availability and durability. Attach up to 16 persistent disks per instance.
View disk docs →
CLI command
Create a persistent disk via the Crusoe CLI:
crusoe storage disks create \ --name data-1 \ --size 100GiB \ --location us-southcentral1-a \ --block-size 4096
Recommended start
Start with a persistent disk
Name, size, and location are required. Use block-size 4096 (default). When attaching to a VM, the disk must be in the same location. Use Instance Templates to provision VMs with pre-attached disks.

Design Directions
Three approaches to empty states — compare Mild, Medium, and Spicy for 5 key surfaces
Mild
Delightful & Focused
Clean, illustration-led. One job: get the user to the primary action.
  • Large centered illustration (SVG or Lottie)
  • Single short headline (Space Mono)
  • One-line supporting blurb — optional
  • ONE primary CTA button only
  • No CLI block, no doc cards, no steps
  • Generous whitespace throughout
Medium
Informational & Helpful
Current pattern. Hero + context cards guide advanced users quickly.
  • Hero card with icon + headline + blurb
  • Primary + secondary CTA
  • Recommended start tip (info chip)
  • 2–3 action cards (Docs, CLI, Quickstart)
  • CLI code snippet with copy button
  • Dense but structured layout
Spicy
Bold & Motion-forward
Dark canvas + animation. Requires collaboration with brand & marketing.
  • Dark gradient / animated canvas background
  • Large display headline (Space Mono, white)
  • Short punchy sub-line
  • Single high-contrast CTA button
  • Motion / particle system (Lottie / CSS)
  • Needs brand sign-off before shipping
